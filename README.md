## 💡 Dataset Structure and Download

You can download intent-based semantic embedding files in the following datasets:

**Amazon-book**/ **Yelp**/ **Amazon-movie** [[Google Drive]](https://drive.google.com/file/d/18gt8SNI2gpTJE5pshZ69Cy_EnqLrpY1X/view)

Each dataset consists of a training set, a validation set, and a test set. During the training process, we utilize the validation set to determine when to stop the training in order to prevent overfitting.

```plaintext
- book (yelp/movie)
|--- trn_mat.pkl # training set (sparse matrix)
|--- val_mat.pkl # validation set (sparse matrix)
|--- tst_mat.pkl # test set (sparse matrix)
|--- usr_emb_np.pkl # user text embeddings
|--- itm_emb_np.pkl # item text embeddings
|--- user_intent_emb_3.pkl # user intent embeddings
|--- item_intent_emb_3.pkl # item intent embeddings
```

## Offline item-intent prototype clustering

To initialize later intent prototypes from the item-intent embeddings, run the
offline IACLR-style KMeans script:

```bash
python encoder/cluster_item_intents.py \
  --input data/amazon/item_intent_emb_3.pkl \
  --output data/amazon/item_intent_prototypes_128.npy \
  --labels-output data/amazon/item_intent_cluster_ids_128.npy \
  --num-clusters 128
```

The prototype file has shape `[num_clusters, intent_embedding_dim]`. By default,
the script follows IACLR with `random_state=2024`, `n_init=10`, and no extra
normalization. For a lower-memory approximation, add `--method minibatch`.

## User-group datasets

To split Amazon-movie users by their number of training interactions, run:

```bash
python encoder/split_user_groups.py --dataset movie
```

This creates three independently trainable datasets under `data/`:
`movie_u1_5`, `movie_u6_10`, and `movie_u11_plus`. User-side embeddings are
split with the interaction matrices, the item ID space is retained, and each
directory contains `original_user_ids.npy` for mapping compact user IDs back to
the original Amazon-movie IDs. Unchanged item embeddings use hard links by
default to save disk space; pass `--copy-item-embeddings` for standalone copies.

Train the groups separately by changing `--dataset`:

```bash
python encoder/train_encoder.py --model lightgcn_int --dataset movie_u1_5 --cuda 0
python encoder/train_encoder.py --model lightgcn_int --dataset movie_u6_10 --cuda 0
python encoder/train_encoder.py --model lightgcn_int --dataset movie_u11_plus --cuda 0
```

The grouped datasets inherit the `movie` model hyperparameters through their
`split_meta.json` metadata while keeping group-specific log and checkpoint
names.
